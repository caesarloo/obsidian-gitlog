# Obsidian Git Log 插件需求文档

## 1. 术语定义

- **Git插件**：指Obsidian的Git插件，用于在Obsidian中管理Git版本控制
- **Copilot插件**：指Obsidian的Copilot插件，提供AI辅助功能
- **大模型**：指Copilot插件使用的人工智能模型，用于生成文本内容
- **Git差异**：指Git仓库中文件的变更内容，包括新增、修改和删除
- **提交日志**：指Git提交时的消息，描述本次提交的内容和目的

## 2. 项目概述

开发一套Obsidian插件，依赖Git插件和Copilot插件，使用Copilot插件的大模型能力，分析Git文件差异，自动生成提交日志，并填充到Git的提交消息中。本插件旨在提高用户的Git提交效率，减少手动编写提交日志的工作量，同时提高提交日志的质量和一致性。

## 3. 功能需求

### 3.1 用户故事

1. **作为Obsidian用户**，我希望能够自动生成Git提交日志，以便节省手动编写日志的时间
2. **作为开发者**，我希望生成的提交日志能够准确反映代码变更，以便提高代码可维护性
3. **作为团队成员**，我希望提交日志格式一致、内容清晰，以便更好地理解代码历史
4. **作为Git用户**，我希望能够在生成的日志基础上进行编辑，以便添加个性化内容

### 3.2 核心功能

1. **Git差异分析**：
   - 检测当前Obsidian仓库的Git文件变更
   - 分析文件的增删改操作和具体内容变化
   - 支持分析多个文件的变更

2. **智能日志生成**：
   - 调用Copilot插件的大模型API
   - 基于文件差异内容生成有意义的提交日志
   - 支持中文和英文两种语言的日志生成
   - 生成符合Git提交规范的日志格式

3. **提交消息填充**：
   - 将生成的日志自动填充到Git提交消息输入框
   - 支持用户编辑和调整生成的日志内容
   - 提供日志预览功能

4. **操作流程优化**：
   - 提供快捷命令触发日志生成
   - 集成到Obsidian的命令面板
   - 支持通过快捷键触发
   - 提供右键菜单选项

### 3.3 辅助功能

1. **配置选项**：
   - 日志语言设置（中文/英文）
   - 日志格式模板选择（默认、简洁、详细）
   - 生成策略调整（详细/简洁）
   - 历史记录保存数量设置

2. **历史记录**：
   - 保存最近生成的日志记录（默认10条）
   - 支持快速重用历史日志
   - 提供历史记录管理界面

3. **错误处理**：
   - 处理Git仓库未初始化的情况
   - 处理Copilot插件未安装的情况
   - 处理大模型API调用失败的情况
   - 提供友好的错误提示和解决方案

## 4. 技术需求

### 4.1 依赖项

- **Obsidian Git插件**：版本 2.20.0 或更高，用于Git操作和差异分析
- **Obsidian Copilot插件**：版本 1.5.0 或更高，提供大模型API访问
- **Node.js**：版本 18.0.0 或更高，插件开发环境
- **TypeScript**：版本 5.0.0 或更高，插件开发语言
- **Obsidian API**：兼容Obsidian 1.5.0 或更高版本

### 4.2 技术实现要点

1. **Git差异获取**：
   - 使用Git插件的API获取文件变更信息
   - 解析Git diff输出，提取变更内容
   - 支持分析单个文件和多个文件的变更

2. **大模型调用**：
   - 构建有效的提示词（Prompt），包含文件差异内容和期望的日志格式
   - 调用Copilot插件的生成API，使用以下参数：
     - model: "gpt-4"（或Copilot插件默认模型）
     - temperature: 0.3（保证生成内容的一致性）
     - max_tokens: 500（足够生成完整的提交日志）
   - 处理API响应和错误，包括超时、限流等情况
   - 实现重试机制，处理临时API失败

3. **用户界面**：
   - 命令面板集成，添加"Generate Git Commit Log"命令
   - 配置界面，提供详细的设置选项
   - 日志预览和编辑界面，支持实时预览生成的日志
   - 右键菜单集成，支持在文件上直接生成日志

4. **性能优化**：
   - 异步处理文件差异分析，避免阻塞主线程
   - 缓存大模型响应，减少重复调用
   - 分批处理大文件变更，提高处理速度
   - 性能指标：
     - 响应时间：生成日志的时间不超过3秒
     - 内存占用：不超过100MB
     - CPU使用率：峰值不超过50%

5. **安全性**：
   - 保护用户的Git仓库信息，不向第三方发送敏感数据
   - 遵守Copilot插件的使用条款和API限制
   - 处理用户隐私数据，确保数据安全

### 7.1 潜在风险

1. **依赖风险**：
   - Git插件或Copilot插件的API变更
   - 大模型API的稳定性和可用性
   - 大模型API的访问权限和费用问题

2. **性能风险**：
   - 大文件变更时的处理时间
   - 大模型调用的响应延迟
   - 网络中断导致的API调用失败

3. **质量风险**：
   - 生成的日志质量不稳定
   - 特殊文件类型的处理
   - 多语言支持的准确性

### 7.2 风险缓解措施

1. **依赖管理**：
   - 版本锁定和兼容性检查
   - 优雅降级和错误处理
   - 定期检查依赖插件的更新

2. **性能优化**：
   - 异步处理和缓存机制
   - 分批处理大文件变更
   - 实现超时机制和重试策略

3. **质量保障**：
   - 多轮提示词优化
   - 人工审核和调整机制
   - 针对不同文件类型的专门处理逻辑

4. **费用管理**：
   - 监控API调用次数
   - 实现本地缓存，减少重复调用

插件的核心价值在于：
- 提高Git提交的效率和规范性
- 减少手动编写提交日志的工作量
- 利用AI技术提升提交日志的质量

通过本插件，用户可以将更多精力集中在实际的内容创作和开发工作上，而不是花费时间思考如何编写规范的提交日志。
