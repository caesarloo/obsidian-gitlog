# Obsidian GitLog 发布说明

## 版本信息
- 版本：`v1.0.2`
- 发布日期：2026-02-27
- 版本类型：功能增强 + 发版流程修复

## 主要更新

### 1) Windows 本地开发环境一键修复
新增 Windows 环境修复脚本，可快速处理本地开发常见环境问题。

- 新增命令：`npm run doctor:win`
- 自动设置 PowerShell 执行策略（`CurrentUser` 范围）
- 自动检查并尝试修复 `uv`、`node`、`npm` 的 PATH 可用性
- 对常见失败场景提供清晰提示（如未安装 Node.js、需重开终端）

### 2) Git 变更路径解析兼容性优化
优化 Git 状态路径处理逻辑，提高复杂路径场景下的稳定性。

- 支持重命名路径（`old -> new`）解析
- 支持 Git 引号与转义路径解码
- 降低包含特殊字符/非 ASCII 路径时的解析失败概率

### 3) Release 资产完整性修复
修复发版后插件安装资产不完整的问题，确保安装所需文件齐全。

- 构建后自动复制：`manifest.json`、`styles.css` 到 `dist`
- 与 `main.js` 一起作为插件安装资产发布
- 减少用户误用源码压缩包（`Source code`）导致的安装失败

## 技术变更清单
- Git 路径解析优化：`src/gitService.ts`
- Release 资产拷贝脚本：`scripts/copy-plugin-assets.mjs`
- Windows 环境修复脚本：`scripts/setup-dev-env.ps1`
- 版本号同步：`package.json`、`package-lock.json`、`manifest.json`

## 验证建议
- 执行：`npm run build`
- 检查 `dist` 目录是否包含：`main.js`、`manifest.json`、`styles.css`
- 若本地命令异常，先执行：`npm run doctor:win`

## 升级说明
- 本次为小版本更新，建议直接升级至 `v1.0.2`。
- 若你依赖旧版构建产物结构，请在升级后确认发布脚本与安装流程。
